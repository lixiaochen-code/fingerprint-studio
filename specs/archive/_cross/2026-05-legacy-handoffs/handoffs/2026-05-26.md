### Phase 1d:扩展代理(完全切换)

**目标**:Chromium 启动不再带 `--proxy-server=`,改由扩展 `chrome.proxy.settings.set` 接管。涉及:

- **NEW** [electron/proxies/controllerExtension.ts](../../electron/proxies/controllerExtension.ts) —— per-profile 写到 `<profile>/auto-registry-proxy-controller-extension/`:
  - manifest:`permissions: ['proxy', 'webRequest', 'webRequestAuthProvider', 'storage']`,host_permissions `<all_urls>`,service worker

- **MOD** [electron/kernel.ts](../../electron/kernel.ts):
  - `buildLaunchArgs` 删 `--proxy-server` 参数
  - `proxy.proxyId === null` → 不加载 controller 扩展(Chromium 回落系统代理)
  - `proxy.proxyId !== null` → 通过 ProxyStore 查 host/port/auth,生成 controller 扩展,追加 `--load-extension`
- **MOD** [electron/main.ts](../../electron/main.ts):
  - `launchProfile` 当前从 `profile.proxy` 取 inline 数据;改成从 ProxyStore 查 proxyId
  - 把 ProxyStore 句柄传给 buildLaunchArgs,或者把 resolved proxy 提前查好再传
- **DELETE** [electron/proxyAuth.ts](../../electron/proxyAuth.ts) + [electron/proxyTest.ts](../../electron/proxyTest.ts):
  - controllerExtension 覆盖 proxyAuth 的 407 处理
  - main.ts 老 `proxy:test` IPC 退役(改成 ProxiesView 用的 `proxies:test`);preload 老 `proxy.test` API 一并清

**风险**:
1. **扩展加载窗口**:扩展加载到 `chrome.proxy.settings.set` 生效有几十毫秒,期间 Chromium 用系统代理。如 startUrl 立即被访问且系统无代理 → 首请求泄真实 IP。**缓解**:Phase 2 的 cdpBootstrap 完成后,startUrl 改成 CDP `Page.navigate` 时机延迟到 bootstrap 就绪后发
2. **应急回滚通道**:`AUTO_REGISTRY_PROXY_VIA=cmdline` env 跳回 `--proxy-server` 老路径(UI 暂不暴露,作为紧急按钮)
3. **kernel.ts 调用方更新**:`buildLaunchArgs` 删 `proxyUrl` 参数会影响调用点(main.ts launchProfile);要同步改

## 11. 风险与待决事项

1. **Phase 1d 启动期代理泄漏**:扩展加载到 `chrome.proxy.settings.set` 生效有几十毫秒窗口,startUrl 若立即被访问会泄真实 IP。Phase 2 cdpBootstrap 通过 `Page.navigate` 延迟 startUrl 缓解
2. **CDP 双 client 兼容性**:cdpBootstrap 与 scripts/runtime puppeteer 共享 browser-level CDP,需要确认 `Target.setAutoAttach` 不抢占;同 target 上多 client attach 在 CDP 协议层允许
3. **schemaVersion 单向**:proxies.json schemaVersion=1 后旧版应用读到这个文件会按 ProxyStore 兼容代码读;但 `BrowserProfile.proxyId` 在旧版应用是 undefined,可能拿不到代理。**结论**:此次升级 forward-only,装了旧版的机器别再开新数据
4. **socks 代理 geo 暂不支持**:proxies/test.ts 对 socks5/socks4 只测 TCP 连通性,不取 geo(需要 socks-proxy-agent 依赖)。Phase 1e 之后如果用户需要可以补
5. **proxyAuth.ts 在 Phase 1d 删除前的退化**:目前两者并存,kernel.ts 还在用 proxyAuth 处理 407;Phase 1d 才会切换到 controllerExtension 统一接管
