## 4. 用户拍板过的关键原则（不要再争论）

1. **逻辑闭环**：主进程是真源，UI 派生。绝不让 UI 自己维护一份会和主进程不一致的状态。
2. **一个 profile 同一时刻最多 1 个活跃 ScriptRun**。GUI 启动的浏览器**不算占用**——脚本可以接管它，但脚本运行期间该 profile 不能被另一个脚本"再占一次"。
3. **startUrl 只在首次启动时打开**。判定方法：`<profilePath>/Default` 是否存在。脚本路径**永远**不开 startUrl（避免抢 page.goto 的 tab）。
4. **业务"平台"概念已删**。`BrowserProfile.platform` 字段不存在；`fingerprint.platform`（navigator.platform 值）保留。
5. **shadcn/ui 唯一 UI 库**（NO antd / MUI）；Tailwind utility；pnpm；lucide-react。
6. **i18n 双语**：所有用户可见文本同时给 en / zh。
7. **原子 JSON 写**：`writeJsonAtomic` + `quarantineCorruptFile`，所有持久化必经。
8. **不主动加测试**，除非用户要。
9. **不留无主 TODO**：要么追到 issue，要么不写。

## 8. 关键技术决策（已固化）

| 决策 | 落位 |
| --- | --- |
| 每个 ScriptRun = 一个 `child_process.fork` | `electron/scripts/runtime.ts` |
| 用户 TS 用 `esbuild.transformSync` 同步转译 | `electron/scripts/bootstrap.ts` |
| `require('auto-registry')` 劫持到 `\0auto-registry-virtual` 虚拟模块 | `electron/scripts/bootstrap.ts` |
| 浏览器启动加 `--remote-debugging-port=0 --remote-debugging-address=127.0.0.1` | `electron/kernel.ts` |
| Monaco 单实例：`loader.config({ monaco })` | `src/components/ScriptEditor.tsx` |
| Monaco 类型源：合并版 ambient d.ts（auto-registry + puppeteer-core 真包 + axios/dayjs/zod/cheerio stub） | `src/lib/scriptTypings.ts` |
| Monaco worker 本地 `?worker` import，不走 CDN | `src/lib/monacoSetup.ts` |
| Vite build：`cross-env NODE_OPTIONS=--max-old-space-size=8192` 防 Monaco OOM | `package.json` script |
| profile 占用：`ScriptRuntimeManager` 每次 start/exit 广播 `'active-changed'`，UI 顶层订阅一次 | `electron/scripts/runtime.ts` + `src/App.tsx` |
| PROFILE_BUSY 错误：`{ code: 'PROFILE_BUSY', occupiedBy: { runId, scriptId } }` | `electron/scripts/runtime.ts` + `electron/main.ts` |
| 代理认证：webRequest 扩展（Chromium M76+ 扔掉 URL 中的 user:pass） | `electron/proxyAuth.ts` |
| 代理探活：裸 socket + HTTP CONNECT www.gstatic.com:443 + 5s 超时 | `electron/proxyTest.ts` |
