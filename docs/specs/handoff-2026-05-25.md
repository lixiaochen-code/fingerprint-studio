## 6. 没做但需要做的事(优先级排序)

### Next 1 — #2 路径之 CDP 引导子系统

**目标**:每次启动 profile,主进程 attach CDP,给每个 tab 同步设置:
- `Network.setUserAgentOverride({ userAgent, userAgentMetadata })` —— `userAgentMetadata` 必须带 brands / platform / platformVersion / architecture / model / mobile,跟 fingerprint payload 对齐
- `Network.setExtraHTTPHeaders({ 'Accept-Language': fingerprint.language })`
- `Page.addScriptToEvaluateOnNewDocument(<stealth payload>)` —— 比 MV3 content_script 更早一个 tick,作为 `world: "MAIN"` 的双保险

**架构提示**:
- 当前 CDP 连接仅在 [electron/scripts/cdp.ts](../../electron/scripts/cdp.ts) 用于脚本子系统(`waitForDevToolsEndpoint`)
- 需要在 [electron/main.ts](../../electron/main.ts) `launchProfile` 内 spawn 之后立即 `waitForDevToolsEndpoint` + 创建一个轻量 CDP 客户端(原生 `ws` 即可,不要拖 puppeteer 进来)
- `Target.setDiscoverTargets({discover:true})` + `Target.setAutoAttach({autoAttach:true, waitForDebuggerOnStart:true, flatten:true})` —— 这套是给所有 tab 装 override 的关键
- 实现完后,[electron/stealth/patches/navigator.ts](../../electron/stealth/patches/navigator.ts) 要加 `navigator.userAgentData` + `getHighEntropyValues` hook(Turnstile 会调)

### Next 2 — fingerprint.ts 版本号对齐

`chromeVersion()` 当前返回 `120-131` 随机。改成:
- 启动时探测真实 Chrome binary 版本(从 `--version` 或 puppeteer/browsers metadata)
- UA 字符串里的 Chrome 版本号 = 真实版本(major / minor 微调可以,major 必须等)
- `userAgentData.brands` 也用同版本

### Next 3 — 指纹组合查表

[fingerprint.ts:42-143](../../electron/fingerprint.ts#L42-L143) 把 `(targetOs, gpu)` → `(hwconcurrency, deviceMemory, screen, fonts)` 从独立随机改成查表,避免"Mac M3 + 4 核 / Windows GTX 1660 + 4 核"这种真机不存在的组合。

### Next 4 — 老的 stealth patch 细调

- [navigator.ts:19](../../electron/stealth/patches/navigator.ts#L19) `webdriver=false` 改为**删除属性**(真实 headful Chrome 是 undefined)
- [chromeRuntime.ts:55](../../electron/stealth/patches/chromeRuntime.ts#L55) `chrome.runtime.id = undefined` —— 非扩展页面 `chrome.runtime` 整个对象不存在,补一个空 runtime 反而是已知的 puppeteer-stealth 指纹
